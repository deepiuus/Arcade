Project Overview

The Arcade project is a gaming platform that allows playing different games with different graphic libraries. Its modular architecture enables dynamic switching between games and graphic renderers during runtime.

Architecture

The project uses a plugin-based architecture with dynamic libraries:

  • Core: Manages game and graphics libraries
  • Graphics Libraries: SFML, SDL2, NCurses
  • Game Libraries: Snake, Nibbler, Menu

Key Components

  • LibraryManager: Handles loading and switching between graphic libraries
  • GameManager: Handles loading and switching between game libraries
  • IGraphics: Interface for all graphic libraries
  • IGames: Interface for all game implementations

Building the Project

The project uses CMake for building. Follow these steps:

Prerequisites

  • C++ compiler with C++17 support
  • CMake 3.10 or higher
  • SFML, SDL2, and NCurses development libraries

Build Commands

mkdir -p build
cd build
cmake ..
make

This will generate the arcade executable and library files in the project directory.

Usage

Starting the Application

./arcade <graphics_library_path>

Example: ./arcade lib/arcade_sfml.so

Controls

Key Action
Arrow keys Move character/selection
Space Select/Restart game
Esc Exit

Graphics Libraries

SFML

A comprehensive multimedia library with rich graphical features.

File: lib/arcade_sfml.so

SDL2

Simple DirectMedia Layer library for low-level access to audio, keyboard, mouse, joystick, and graphics.

File: lib/arcade_sdl2.so

NCurses

Text-based interface library for terminal-based visualization.

File: lib/arcade_ncurses.so

Available Games

Snake

Classic snake game where you control a snake that grows as it eats food while avoiding walls and itself.

File: lib/arcade_snake.so

Nibbler

A snake game variant with maze navigation, where you guide a snake through a predefined map.

File: lib/arcade_nibbler.so

Map: lib/Games/nibbler/map.txt

Menu

Game selection interface to choose between available games.

File: lib/arcade_menu.so

Extending the Project

Adding a New Graphics Library

To create a new graphics library:

  1. Create a new class that implements the IGraphics interface
  2. Implement all required methods (Init, getKey, Display, etc.)
  3. Create an entryPoint function that returns a new instance of your class
  4. Build as a shared library (.so file)

Adding a New Game

To create a new game:

  1. Create a new class that implements the IGames interface
  2. Implement all required methods (setKey, getDisplay, getSound, etc.)
  3. Create an entryPoint function that returns a new instance of your game
  4. Build as a shared library (.so file)
